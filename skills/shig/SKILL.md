# ソシオメディア ヒューマンインターフェース ガイドライン（SHIG）

UI を設計・実装する際に、SHIG の 100 原則を適用すること。
各原則の詳細は [ソシオメディア公式サイト](https://www.sociomedia.co.jp/category/shig) を参照。

> **著作権への配慮**: 本スキルは原則名と URL の一覧のみを掲載し、原文の引用・要約は行わない。
> 各テーマに対する「実装への適用ノート」は独自に作成したものである。

## テーマ一覧と行動指針

### 1. 基本設計原則

**核心**: シンプルにする（#1）、メンタルモデル（#3）、一貫性（#6）

UIを実装するとき → 機能を足す前に「本当に必要か」を問い、ユーザーの既存の知識で操作できる構造にする。プラットフォーム・アプリ内・コンポーネント単位の3層で一貫性を保つ。

### 2. 認知心理学とヒューリスティクス

**核心**: フィッツの法則（#16）、ヒックの法則（#17）、複雑性保存の法則（#18）

UIを実装するとき → 重要なアクションのクリックターゲットを大きくし画面端を活用する。選択肢が多い場合はカテゴリ分けや段階的開示で認知負荷を分散する。

### 3. 知覚と記憶

**核心**: 視覚ゲシュタルト（#10）、ユーザーの記憶に頼らない（#12）、空間記憶（#76）

UIを実装するとき → 近接・類似・連続の法則でグルーピングし、操作に必要な情報はすべて画面上に提示する。要素の位置は一貫させて空間記憶を活用する。

### 4. オブジェクト指向UI

**核心**: オブジェクトベースにする（#23）、名詞→動詞の操作順序（#26）

UIを実装するとき → まずオブジェクト（名詞）を選ばせ、次にアクション（動詞）を提示する。一覧→詳細→編集の自然な流れを設計し、オブジェクトの状態を視覚的に体現する。

### 5. インタラクション設計

**核心**: ユーザーの主導権（#7）、モードレス（#9）、エラーを回避する（#15）

UIを実装するとき → ユーザーが自由にナビゲートでき、いつでも操作を中断・取り消せるようにする。モーダルダイアログは最終手段とし、エラーを起こさせない設計を優先する。

### 6. 操作の効率化と自動化

**核心**: ショートカットを用意する（#22）、自動化する（#29）、黙って実行する（#57）

UIを実装するとき → 頻繁な操作にはキーボードショートカットを提供する。選択肢が1つしかないなら自動実行し、不要な確認ダイアログは省く。

### 7. 入力とフォーム設計

**核心**: ストーリー性（#40）、よいデフォルト（#42）、制限コントロール（#43）

UIを実装するとき → フォームのフィールド順序にストーリーを持たせ、適切なデフォルト値を設定する。自由入力より制限コントロール（ラジオ、セレクト等）を優先して入力エラーを防ぐ。

### 8. プレゼンテーションとフィードバック

**核心**: 0.1秒以内に反応（#65）、操作の近くでフィードバック（#66）、プログレッシブ・ディスクロージャ（#67）

UIを実装するとき → ユーザーの操作に対し即座に視覚的フィードバックを返す。情報は必要な分だけ段階的に開示し、全体を一度に見せない。

### 9. モバイルとタッチ

**核心**: タッチ要素7mm以上（#78）、ドリルダウンは上→下・左→右（#80）

UIを実装するとき → タッチターゲットは最低44px（7mm）を確保する。階層ナビゲーションでは左→右を進む方向、右→左を戻る方向として一貫させる。

### 10. 国際化・アクセシビリティ・デザイン哲学

**核心**: 文化依存しないアイコン（#68）、音声読み上げ対応（#94）、色に依存させない（#96）

UIを実装するとき → 記号やアイコンの文化的解釈の違いを意識する。セマンティックHTMLとARIA属性で支援技術に対応し、色だけでなくテキストや形状でも情報を伝える。

## 設計判断チェックリスト

UIを設計・レビューする際に以下を確認すること：

**構造**
- [ ] オブジェクト（名詞）→アクション（動詞）の順序になっているか
- [ ] ユーザーのメンタルモデルと一致しているか
- [ ] モードレスな構造になっているか（モーダルは最終手段）

**認知負荷**
- [ ] 一度に提示する選択肢は適切な数か（ヒックの法則）
- [ ] ユーザーの記憶に頼っていないか（必要な情報は画面上に表示）
- [ ] 一貫性は保たれているか（プラットフォーム・アプリ・コンポーネント）

**インタラクション**
- [ ] ユーザーはいつでも操作を取り消し・中断できるか
- [ ] エラーを回避する設計になっているか（制限コントロール、バリデーション）
- [ ] フィードバックは即座に、操作の近くで表示されるか

**フォーム**
- [ ] フィールドの順序にストーリー性があるか
- [ ] 適切なデフォルト値が設定されているか
- [ ] 制限コントロールを使えるところで自由入力にしていないか

**プレゼンテーション**
- [ ] 視覚的階層は明確か（ゲシュタルト原則）
- [ ] アニメーション/トランジションは状態変化を理解する助けになっているか
- [ ] プログレッシブ・ディスクロージャーが適用されているか

**アクセシビリティと国際化**
- [ ] タッチターゲットは十分な大きさか（44px / 7mm 以上）
- [ ] 色に依存せず情報を伝えているか
- [ ] 音声読み上げに対応しているか（セマンティックHTML、ARIA）
- [ ] 文化依存のアイコンや記号を使っていないか
