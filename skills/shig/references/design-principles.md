## 実装への適用ノート

### 認知負荷を減らすための設計判断基準

UIの認知負荷を評価する際は、以下の3つの観点から検討する：

**1. 本質的な複雑性と付随的な複雑性の区別**

複雑性保存の法則（#18）により、タスク固有の複雑性は消せない。UIが担うべきは「付随的な複雑性」の排除である。

| 判断基準 | 対応 |
|---------|------|
| ユーザーが理解する必要がある概念か？ | → 残す（ただし段階的に提示） |
| システムの都合で生じた複雑性か？ | → UIが吸収する（自動化、デフォルト値） |
| 専門用語が必要か？ | → ユーザーの言葉に置き換える（#11） |

**2. 記憶の負荷を減らす**

- 操作に必要な情報はすべて画面上に表示する（#12）
- 選択肢を列挙してユーザーに思い出させるのではなく選ばせる
- 前のステップで入力した情報を再入力させない
- 未完了タスクの手がかりを画面上に残す（#77 プロスペクティブメモリー）

**3. 知覚の負荷を減らす**

- ゲシュタルト原則（近接・類似・連続・閉合）でグルーピングする（#10）
- 錯視を考慮してスペーシングやアラインメントを視覚的に調整する（#75）
- 要素の空間的配置を一貫させて位置記憶を活用する（#76）

### 一貫性のレイヤー

一貫性（#6）は3つのレイヤーで管理する：

| レイヤー | 範囲 | 具体例 |
|---------|------|--------|
| プラットフォーム一貫性 | OS・ブラウザの慣例 | スクロール方向、右クリックメニュー、戻るボタン、ショートカットキー |
| アプリ内一貫性 | アプリ全体のパターン | ナビゲーション構造、カラーシステム、アクションの配置ルール |
| コンポーネント一貫性 | 同種の要素 | ボタンのサイズ・色・ラベル規則、フォームの入力パターン |

**優先順位**: プラットフォーム > アプリ内 > コンポーネント

独自の操作体系を発明するよりも、ユーザーが慣れ親しんだ慣用的なパターンを採用する（#71）。「直観的」は個人差が大きいが、「慣用的」はプラットフォームの慣例として共有されている。

### フィッツの法則のWebアプリでの適用

フィッツの法則（#16）：ターゲットに到達する時間は、距離が短いほど・サイズが大きいほど短い。

**Webアプリでの実践**:

| 場面 | 適用方法 |
|------|---------|
| プライマリアクション | ボタンを大きく、目立つ位置に配置 |
| 破壊的アクション | 頻繁に使うアクションから距離を置く |
| ナビゲーション | 画面端（上・左）を活用（端は無限のターゲット幅を持つ） |
| コンテキストメニュー | 操作対象の近くに表示（右クリック、インラインアクション） |
| フォーム送信 | 送信ボタンは最後の入力欄の近くに配置 |

### ヒックの法則のWebアプリでの適用

ヒックの法則（#17）：選択肢の数が増えると、判断にかかる時間が対数的に増加する。

**Webアプリでの実践**:

| 選択肢の数 | 対応 |
|-----------|------|
| 2〜5 | そのまま並列に表示（ラジオボタン、タブ等） |
| 6〜15 | カテゴリ分けまたはセレクトメニュー |
| 16〜50 | 検索可能なセレクト、フィルタ付きリスト |
| 50以上 | インクリメンタルサーチ、オートコンプリート |

**注意**: ヒックの法則は「選ぶ」行為に適用される。スキャンして探す場合（メニュー項目を読む等）は別の考慮が必要。

### シグニファイアとマッピングの実装

**シグニファイア**（#4）：操作可能であることを視覚的に示す手がかり。

| 要素 | シグニファイア | アンチパターン |
|------|--------------|---------------|
| ボタン | 背景色・影・ボーダー、ホバーで変化 | テキストリンクと区別がつかないフラットなボタン |
| リンク | 下線または色の差異、ホバーで変化 | 本文テキストと同じスタイルのリンク |
| ドラッグ可能要素 | グリップハンドル、`cursor: grab` | アイコンなしでドラッグ可能なことが不明 |
| 入力欄 | ボーダー、背景色の差異 | 読み取り専用テキストとの区別がない入力欄 |

**マッピング**（#5）：操作と結果の対応関係を自然にする。

- スライダーを右に動かす → 値が増える
- 上にスワイプ → コンテンツが上にスクロール
- トグルを右にスライド → 機能がオン
- リスト項目をドラッグして並べ替え → 表示順の直接操作
