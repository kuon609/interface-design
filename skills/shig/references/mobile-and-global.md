## 実装への適用ノート

### タッチターゲットサイズの基準

タッチ操作する要素の大きさは 7mm 以上にする（#78）。ホットスポットを広げる（#93）で操作領域をさらに拡張する。

**プラットフォーム別の最小サイズ**:

| プラットフォーム | 最小サイズ | ピクセル（標準密度） |
|---------------|-----------|------------------|
| Apple HIG | 44×44 pt | 44×44 px（@1x） |
| Material Design | 48×48 dp | 48×48 px |
| WCAG 2.5.8 | 24×24 CSS px | 24×24 px（最低限） |
| SHIG | 7mm以上 | 約44px（160dpi基準） |

**ホットスポットの拡張**:

視覚的なサイズが小さくても、タッチ可能な領域（ヒットエリア）を拡張する。

```css
/* 視覚的に小さいアイコンボタンのヒットエリア拡張 */
.icon-button {
  width: 24px;       /* 視覚的サイズ */
  height: 24px;
  padding: 12px;     /* ヒットエリア拡張 → 実効48×48px */
  margin: -12px;     /* レイアウトへの影響を相殺 */
}
```

**タッチターゲット間の間隔**:

- 隣接するタッチターゲットの間に最低 8px のギャップを確保する
- ❌ 誤タップしやすい密集した小さなボタン群
- ✓ 十分な間隔で配置された適切なサイズのボタン

### 記号の国際的な解釈の違い

◯✕△等の記号を安易に使わない（#70）。記号の意味は文化によって異なる。

| 記号 | 日本での意味 | 他の文化での解釈 |
|------|------------|----------------|
| ◯（マル） | 正解、OK、良い | 欧米では特に意味がない、またはゼロ |
| ✕（バツ） | 不正解、NG | 欧米ではチェックマーク（正解）の場合がある |
| △（サンカク） | 注意、微妙 | 多くの文化で特定の意味を持たない |
| ✓（チェック） | 確認済み | ほぼ世界共通で「正しい」「完了」 |

**安全な代替手段**:

| 避けるべき表現 | 国際的に安全な代替 |
|-------------|-----------------|
| ◯ / ✕ で可否を表現 | ✓ / — または「可」「不可」テキスト |
| △ で注意を表現 | ⚠ 警告アイコン + テキスト |
| 色だけで状態を表現 | アイコン + テキスト + 色 の組み合わせ |

**アイコンのモチーフ**（#68）:

- メールボックス → 文化によって形が異なる（日本の郵便ポスト vs 米国のメールボックス）
- 通貨記号 → ¥ / $ / € は地域依存
- ジェスチャー → 親指を立てるサインは地域によって不快
- 宗教的シンボル → 十字、星、三日月は避ける

### ドリルダウンの方向性ルール

ドリルダウンは上→下、左→右（#80）。左が戻るで右が進む（#81）。

**情報階層のナビゲーション方向**:

```
概要（上位）
  │
  ↓  下方向にドリルダウン
  │
詳細（下位）

一覧（左） → 詳細（右） → 編集（さらに右）
       ←  戻る  ←
```

**モバイルでの実装**:

| 操作 | 方向 | アニメーション |
|------|------|-------------|
| 詳細に進む | 左→右 | 新しい画面が右からスライドイン |
| 一覧に戻る | 右→左 | 現在の画面が右にスライドアウト |
| サブメニューを開く | 左→右 | 右からスライドイン |
| メニューを閉じる | 右→左 | 左にスライドアウト |
| ドリルダウン | 上→下 | 下からスライドイン（ボトムシート等） |
| 閉じる | 下→上 | 下にスライドアウト |

**モバイルでは包括的より階層的に**（#82）:

- デスクトップ: ダッシュボード、スプリットビュー等の包括的レイアウト
- モバイル: 一覧→詳細の階層的レイアウト（画面が小さいため）
- タブレット: 画面幅に応じてスプリットビューと階層的レイアウトを切り替え

### アクセシビリティの実装

**音声読み上げ対応**（#94）:

| 要素 | 必須対応 |
|------|---------|
| 画像 | 意味のある `alt` テキスト（装飾画像は `alt=""` ） |
| アイコンボタン | `aria-label` でアクションを説明 |
| フォーム | `<label>` と `<input>` を `for`/`id` で関連付け |
| 動的コンテンツ | `aria-live` で更新を通知 |
| ナビゲーション | ランドマーク（`<nav>`, `<main>`, `<aside>`）を使用 |
| モーダル | `role="dialog"` + `aria-modal="true"` + フォーカストラップ |

**文字の拡大対応**（#95）:

- フォントサイズは `rem` で指定する（`px` ではなく）
- コンテナのサイズは固定しない（`min-height` を使い `height` で固定しない）
- テキストが200%に拡大されてもレイアウトが崩れないことを確認する
- 横スクロールが必要にならないようにする

**色に依存させない**（#96）:

- 色だけで情報を伝えない（アイコン、テキスト、パターンを併用）
- WCAG AA 基準のコントラスト比を満たす（通常テキスト: 4.5:1、大テキスト: 3:1）
- フォーカスインジケーターは色だけでなく形状（アウトライン）でも示す

**ユーザーのペース**（#97）:

- 自動的に消えるトースト通知に重要な情報を載せない（または十分な表示時間を確保）
- タイムアウトがある場合は延長手段を提供する
- 自動再生するコンテンツには停止手段を用意する
- スライドショーやカルーセルは自動送りしない

### 多言語化への配慮

多言語化を想定したUIではラベルの長さの違いを考慮する（#69）。

| 言語 | 英語との文字数比率（目安） |
|------|----------------------|
| 日本語 | 0.5〜0.8倍（文字幅は約2倍） |
| ドイツ語 | 1.3〜1.8倍 |
| フランス語 | 1.2〜1.5倍 |
| 中国語 | 0.5〜0.8倍（文字幅は約2倍） |
| アラビア語 | 0.8〜1.2倍（RTLレイアウト） |

**設計上の注意**:

- ボタンやラベルの幅を固定しない（テキストに応じて伸縮させる）
- テキストの折り返しを許容するレイアウトにする
- アイコン + テキストのボタンでは、テキストが長くなっても崩れないようにする
- RTL（右から左）レイアウトへの対応を考慮する（`direction` プロパティ、論理プロパティ）

**肯定／否定ボタンの順序**（#72）:

| プラットフォーム | 順序 |
|---------------|------|
| macOS / iOS | [キャンセル] [OK] （肯定が右） |
| Windows | [OK] [キャンセル] （肯定が左） |
| Web（一般的） | プライマリアクションを右に配置する傾向 |
| Material Design | [キャンセル] [OK] （肯定が右） |

→ Webアプリではプラットフォームの慣例に従い、一貫した順序を保つこと。
