## 実装への適用ノート

### フィードバックの位置と即時性

操作の近くでフィードバックする（#66）。ユーザーの視線移動を最小限にし、操作と結果の因果関係を明確にする。

**フィードバック位置の原則**:

| 操作 | フィードバック位置 | ❌ 避けるべき位置 |
|------|------------------|------------------|
| ボタンクリック | ボタン上（ローディング表示）→ 操作結果を近くに表示 | 画面端の離れたトースト通知のみ |
| フォーム入力 | フィールドの直下にインラインメッセージ | ページ上部にまとめてエラー表示のみ |
| インライン編集 | 編集箇所の直近に保存状態を表示 | ステータスバー（画面下部） |
| ドラッグ＆ドロップ | ドラッグ中のゴースト要素 + ドロップ先のハイライト | 操作結果を別の画面で確認 |
| 削除 | 削除された場所にUndo付きトースト | 離れた場所の通知 |

**ペンは紙の近くに置く**（#30）：操作を起動するUI要素は、操作対象の近くに配置する。

```
✓ 正しい配置
┌────────────────────────┐
│ タイトル        [編集]  │  ← タイトルの横に編集ボタン
│ 内容テキスト...        │
└────────────────────────┘

❌ 悪い配置
[編集]                        ← ツールバーの遠い位置に編集ボタン
┌────────────────────────┐
│ タイトル               │
│ 内容テキスト...        │
└────────────────────────┘
```

### アニメーション/トランジションの適用基準

画面の変化をアニメーションで表す（#63）。ただし装飾的なアニメーションは避け、状態変化の理解を助けるものだけを使う。

**アニメーションを使うべき場面**:

| 場面 | 目的 | 推奨デュレーション |
|------|------|------------------|
| 要素の出現/消失 | 変化の発生箇所に注意を向ける | 150〜250ms |
| 画面遷移 | ナビゲーションの方向性を示す | 200〜350ms |
| 状態変化 | 変化の過程を見せて理解を助ける | 150〜300ms |
| レイアウトシフト | 要素の移動を追跡可能にする | 200〜300ms |
| ローディング | 処理中であることを伝える | 継続的 |

**アニメーションを使わないべき場面**:

- ユーザーの操作をブロックする長いアニメーション
- 繰り返し表示される要素（毎回アニメーションすると煩わしい）
- 装飾目的のみのアニメーション
- `prefers-reduced-motion` が設定されている場合

**トランジションは両方向につける**（#64）:

- ドロワーが右からスライドインするなら、右にスライドアウトする
- アコーディオンが展開するアニメーションがあるなら、折りたたみにもアニメーションをつける
- ページを右に遷移したら、戻るときは左に遷移する

### 0.1秒ルールの実装

ユーザーの操作に対して0.1秒以内に反応を返す（#65）。

**応答時間の基準**:

| 時間 | ユーザーの知覚 | 必要な対応 |
|------|-------------|-----------|
| 0〜100ms | 即座の反応 | 視覚フィードバック（ホバー、クリック反応） |
| 100〜1000ms | 処理中と感じる | スピナーまたは進捗表示 |
| 1〜10秒 | 待たされていると感じる | プログレスバー + キャンセル手段 |
| 10秒以上 | 離脱の危険 | バックグラウンド処理 + 通知 |

**実装のポイント**:

- ボタンクリック → 即座に押下状態の視覚変化（`:active` 疑似クラス）
- API呼び出し → ボタンをローディング状態に変更（0.1秒以内）
- リスト操作 → 楽観的更新で即座にUIに反映
- ページ遷移 → ルーターレベルでローディングインジケーター表示

**ガッツを見せる**（#58）：システムの内部状態を透明にする。処理中に何が起きているかをユーザーに伝える。

- 「読み込み中...」ではなく「データを取得しています（3/10件）」
- プログレスバーで具体的な進捗を表示
- 完了予測時間を表示（大量データの処理時）

### プログレッシブ・ディスクロージャーのUIパターン

プログレッシブ・ディスクロージャ（#67）：情報を段階的に開示し、ユーザーの認知負荷を管理する。

| パターン | 使用場面 | 実装 |
|---------|---------|------|
| アコーディオン | FAQ、設定項目のグルーピング | クリックで展開/折りたたみ |
| 「もっと見る」ボタン | 長いリストやコンテンツ | クリックで追加コンテンツを読み込み |
| ツールチップ | 補足情報、略語の説明 | ホバーで表示（タッチ対応必須） |
| 詳細パネル/ドロワー | オブジェクトの詳細情報 | 選択で横からスライドイン |
| タブ | 同一コンテキストの情報グルーピング | クリックでパネル切り替え |
| ステッパー | 複雑な入力プロセス | 段階的にフォームを表示 |

**設計の原則**:

- **最初に見せるもの**: ユーザーの80%が必要とする情報
- **次に見せるもの**: 残りの20%が必要とする詳細
- **隠すもの**: ほぼ使われない高度な設定

**スクロール画面では続きがありそうに見せる**（#52）:

- コンテンツがビューポートの境界で見切れるようにし、スクロール可能であることを暗示する
- リストの最後の項目を半分だけ見せる
- 「スクロールしてください」とテキストで伝えるのは避ける

### ナビゲーションとラベリング

**ナビゲーション項目は名詞にする**（#33）:

| ❌ 動詞ベース | ✓ 名詞ベース |
|-------------|-------------|
| 設定する | 設定 |
| メッセージを送る | メッセージ |
| レポートを見る | レポート |
| プロジェクトを管理する | プロジェクト |

**アイコンは名詞または形容詞を表す**（#34）:

- アイコン単体で使用しない（テキストラベルを併記する）
- アイコンが表すのは「もの（名詞）」や「状態（形容詞）」であり、「動作（動詞）」ではない
- 例外：ゴミ箱アイコン（削除）のように十分に慣用化したものは許容

**ウェイファインディング**（#59）：ユーザーが現在地を把握できるようにする。

- パンくずリスト
- アクティブなナビゲーション項目のハイライト
- ページタイトルの一貫した表示
- URLの構造がナビゲーション構造と一致する

---

### ハイライトと視覚的安定性

**ハイライト表現は構成要素をひとつだけ変化させる**（#74）:

| ❌ 過剰なハイライト | ✓ 適切なハイライト |
|-------------------|-------------------|
| 色 + 太字 + 下線 + サイズ変更 | 背景色のみ変更 |
| アイコン + 色 + アニメーション | アイコンのみ追加 |
| ボーダー + 影 + 背景色 | 背景色のみ変更 |

**メニュー項目の位置を変化させない**（#73）:

- メニューの並び順は固定する
- 使用頻度による自動並べ替えは行わない
- 非表示の項目は無効化状態で残す（位置のズレを防ぐ）
