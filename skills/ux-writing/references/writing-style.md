## 実装への適用ノート

### 冗長表現の排除パターン

UIテキストでは「短く、正確に」を原則とする。以下は日本語UIで頻出する冗長表現とその改善例。

| 冗長な表現 | 簡潔な表現 | 理由 |
|-----------|-----------|------|
| ○○することができます | ○○できます | 「することが」が不要 |
| ○○を行なう | ○○する | 「を行なう」は回りくどい |
| ○○についての設定 | ○○の設定 | 「について」が不要 |
| ○○の方 | ○○ | 「の方」は不要な婉曲表現 |
| ○○という名前 | ○○ | 「という」が不要な場合が多い |
| ○○していただく | ○○してください | UIでは丁寧すぎる敬語を避ける |
| ○○させていただきました | ○○しました | 過剰な謙譲表現 |

**判断基準**: 助詞や補助動詞を取り除いても意味が変わらないなら、取り除く。

### 敬語レベルの使い分け

UIの文脈に応じた敬語レベルの目安。情報伝達の効率を最優先する。

| UI要素 | 推奨レベル | 例 |
|--------|-----------|-----|
| ボタンラベル | 常体（敬語なし） | 「保存」「削除」「追加」 |
| 説明文（ディスクリプション） | 敬体（です/ます） | 「○○を設定できます」 |
| エラーメッセージ | 敬体（です/ます） | 「○○が入力されていません」 |
| 確認ダイアログ | 敬体（です/ます） | 「削除してもよいですか？」 |
| ツールチップ | 敬体 or 体言止め | 「○○を設定します」「○○の設定」 |
| プレースホルダー | 常体 or 体言止め | 「例: yamada@example.com」 |

**避けるべき表現**:
- 「ご確認いただけますようお願いいたします」→「確認してください」
- 「○○させていただきます」→「○○します」
- 「○○してくださいませ」→「○○してください」

### 文法の正確性チェック

UIテキストで見落としがちな文法的な誤りのパターン。

| 誤用 | 正しい表現 | 解説 |
|------|-----------|------|
| ご参考ください | ご参考にしてください | 「参考」はサ変動詞ではない |
| ご利用ください | ✓（正しい） | 「利用する」はサ変動詞なので「ご〜ください」可 |
| ○○が存在しない場合 | ○○がない場合 | UIでは平易な表現を優先 |
| データの取得に失敗 | データを取得できません | ユーザー視点の表現にする |

**「ご○○ください」の判断基準**: 「○○する」とサ変動詞にできる場合のみ「ご○○ください」が使える（ご利用ください、ご確認ください）。「する」を付けられない名詞には「ご○○にしてください」を使う。

### 常用漢字と表外読みの判断

常用漢字表にない読み方（表外読み）はUIテキストに使わない。

| 表外読み（避ける） | 推奨表記 |
|------------------|---------|
| 想い | 思い |
| 旧い | 古い |
| 繋がる | つながる |
| 拘る | こだわる |
| 辿る | たどる |
| 溢れる | あふれる |
| 捗る | はかどる |

**参考**: [文化庁 常用漢字一覧表](https://www.bunka.go.jp/kokugo_nihongo/sisaku/joho/joho/kakuki/14/pdf/jyouyou_kanjihyou.pdf)

### カタカナ語と漢字語の選択基準

カタカナ語は受け手によって意味の解釈幅が大きいため、漢字語で正確に表現できる場合は漢字語を優先する。

| カタカナ語 | 漢字語の候補 | 判断 |
|-----------|------------|------|
| エビデンス | 証拠、根拠 | 漢字語を優先 |
| コンセンサス | 合意 | 漢字語を優先 |
| リソース | 資源、人材 | 文脈による |
| ステータス | 状態、状況 | UIでは「ステータス」が慣用化しているため許容 |
| ダウンロード | — | 定着したIT用語はカタカナのまま |
| フィルター | — | 定着したIT用語はカタカナのまま |

**判断基準**: 対象ユーザーの大多数が意味を正確に共有できるカタカナ語はそのまま使う。解釈のブレが生じやすいカタカナ語は漢字語に置き換える。IT業界で十分に定着した用語（ダウンロード、フィルター、ログイン等）はカタカナのまま使用する。

### 用語統一の判断フロー

同じ概念に対して複数の表現候補がある場合の判断手順。

```
1. プロダクト内に既に使われている表現はあるか？
   ├─ Yes → 既存の表現に合わせる
   └─ No → 次へ

2. ユーザーが日常的に使う言葉か？
   ├─ Yes → その言葉を採用する
   └─ No → 次へ

3. 辞書に載っている一般的な言葉か？
   ├─ Yes → その言葉を採用する
   └─ No → より一般的な言い換えを検討する
```

**原則**: 一度決めた表現はプロダクト全体で統一する。意図的に言い換えるのはヘルプ文書等でユーザーの理解を助ける場合のみ。
