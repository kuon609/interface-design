# ビジュアルデザイン リファレンス

## スペーシングシステム

### スケール

以下の値のみ使用する（base-8）：`4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 48 · 64 · 80 · 96px`

実装例：
```css
/* CSS カスタムプロパティ */
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
--space-8: 32px;
--space-10: 40px;
--space-12: 48px;
--space-16: 64px;
```

Tailwind では `p-1`、`p-2`、`p-3`、`p-4`、`p-5`、`p-6`、`p-8`、`p-10`、`p-12`、`p-16` に対応。

### グルーピングの法則

関連度が高いほど近く、関連度が低いほど遠く配置する：
- ラベル → 入力欄：`4〜8px`
- フォームグループ内のフィールド間：`12〜16px`
- フォーム内のセクション間：`24〜32px`
- 独立したカードやセクション間：`32〜48px`
- ページセクション間：`48〜80px`

### 使ってはいけない値

任意の値：`7px`、`11px`、`13px`、`17px`、`23px`、`25px`、`30px`、`35px` など。
これらは「深く考えていない」ことを示し、ページ全体に視覚的なノイズを生む。

## 視覚的階層

### 3レベルルール

一度に表示するテキスト階層は最大 3 レベルに留める：
1. **プライマリ** — ページタイトル、セクション見出し、主要指標
2. **セカンダリ** — サブ見出し、カードタイトル、フィールドラベル
3. **ターシャリ** — 本文、説明文、メタデータ

4 番目が必要なときは段階的開示（折り畳み、タブ、詳細ビュー）を使う。

### 階層のシグナル（最低 2 つ組み合わせる）

| シグナル | 方法 |
|---------|------|
| サイズ | 大きい＝重要 |
| ウェイト | 太い＝重要 |
| カラー | 高コントラスト＝重要 |
| スペーシング | 余白が多い＝重要 |
| 位置 | 左上から読み始める |

サイズだけ（同じウェイト、同じカラー）→ 弱い階層。
ウェイトだけ（同じサイズ、同じカラー）→ 弱い階層。

### プライマリアクションは 1 つ

各画面に視覚的に目立つプライマリアクションは 1 つだけ。サポートアクションは：
- サイズを小さく
- コントラストを下げる（セカンダリ/アウトライン/テキストボタン）
- 読み始めの位置から離す

同等の重みを持つ 2 つのプライマリアクションを並べると意思決定の麻痺を招く。

## カラーシステム

### セマンティックロール（必須）

design token システムで以下を全て定義する。省略しない。

```css
/* トークン定義の例 */
--color-primary: ...;         /* メインブランド/アクション */
--color-primary-hover: ...;   /* プライマリのホバー状態 */
--color-secondary: ...;       /* セカンダリアクション */
--color-destructive: ...;     /* 削除・取り消し・危険 */
--color-success: ...;         /* 確認・完了 */
--color-warning: ...;         /* 注意・回復可能な問題 */
--color-error: ...;           /* 失敗状態・バリデーション */
--color-muted: ...;           /* 無効化・プレースホルダー・補足情報 */
--color-surface: ...;         /* カード・パネルの背景 */
--color-border: ...;          /* ボーダー・区切り線 */
--color-text: ...;            /* プライマリテキスト */
--color-text-secondary: ...;  /* セカンダリ・補足テキスト */
```

### カラー + アイコン/テキストのルール

状態やステータスの区別に**カラーのみ**を使わない。
必ずアイコン、ラベル、パターンのいずれかと組み合わせる。

```
❌  エラーフィールドに赤いボーダーだけ（カラーのみ）
✓   赤いボーダー + エラーアイコン + エラーメッセージテキスト
```

これはアクセシビリティ（色盲対応）と UX（コンテキスト伝達）の両方で必要。

### コントラスト最低基準

| コンテキスト | 最低比率 |
|-----------|---------|
| 背景上の本文テキスト | 4.5:1 |
| 大きなテキスト（18px 以上、または 14px 以上 bold） | 3:1 |
| UI コンポーネントとアイコン | 3:1 |
| 装飾的な要素 | 制限なし |

テキストと背景色の組み合わせを決める前にコントラストチェッカーで確認する。

## タイポグラフィ

### スケール

モジュラースケールを使う。よく使われる比率：1.25（Major Third）または 1.333（Perfect Fourth）。

スケール例（比率 1.25、ベース 16px）：
```
xs:   12px
sm:   14px
base: 16px
lg:   20px
xl:   24px
2xl:  32px
3xl:  40px
4xl:  48px
```

使用するサイズは 6 種類以内に統合する。それ以上は混乱を招く。

### ウェイト

装飾ではなく、重要度を伝えるためにウェイトを使う：
- `400`（regular）— 本文、説明
- `500`（medium）— ラベル、サブ見出し
- `600`（semibold）— カードタイトル、セクション見出し
- `700`（bold）— ページタイトル、強調

UI テキストで `300`（light）や `900`（black）は読みやすさを損なうため使わない。

### 行高（Line Height）

| 用途 | 行高 |
|-----|------|
| 見出し | 1.1〜1.3 |
| 本文テキスト | 1.4〜1.6 |
| UI ラベル、単一行 | 1.0〜1.2 |

### 読みやすい行長

段落や長文テキスト：1 行 60〜80 文字（`max-width: 60ch`〜`75ch`）。
データテーブルや UI ラベル：max-width 制限なし。

### 避けること

- 4 語以上の全大文字（読みやすさが大幅に低下）
- UI 文脈で数語以上のイタリック
- リンク以外へのアンダーライン
- 1 つのインターフェース内で 2 種類以上のフォントファミリー

## レイアウト

### グリッド

カラムグリッドを決めて全コンテンツをそれに沿わせる。よく使われる選択：
- ダッシュボードや複雑なレイアウト：12 カラムグリッド
- シンプルなアプリ：4 または 6 カラム
- フォーム：最大幅付きシングルカラム

意図なくグリッドを崩さない。崩す場合（ヒーロー画像、全幅セクション）は意図的かつ一貫して行う。

### アライメント

- 基本は左揃え（テーブル内の数値は右揃え）
- 見えないグリッドラインに沿って要素を配置する
- 2 行以上のテキストブロックは中央揃えにしない
- ラベルと入力欄のアライメントは常に統一（左揃えか右揃えか、混在させない）

### レスポンシブブレークポイント

標準的なブレークポイント：
```
sm:  640px  （横向きスマートフォン）
md:  768px  （タブレット）
lg:  1024px （ラップトップ）
xl:  1280px （デスクトップ）
2xl: 1536px （ワイドスクリーン）
```

モバイルファーストで設計する。大きい画面に向けて複雑さを追加し、小さい画面で削減する方向はとらない。
